TabCordion
A hybrid UI component that functions as an accordion on small screens and transforms into tabs on larger viewports (≥700px). Built with semantic <details> and <summary> elements, enhanced with Container Queries and Anchor Positioning.
Accordion
The default state on narrow viewports. Items stack vertically with expandable content. Use the from attribute to control styling with tokens like background, divided, rounded, or breakout for visual effects.
Tabs
Activated by adding a to attribute on viewports ≥700px. Transforms the accordion into a horizontal tabbed interface. Style with tokens like highlight, line, compact, or shadow for different visual treatments.
From Tokens
Control accordion appearance: background adds padding and color, contain wraps items in a border, divided adds separators, separate creates cards, breakout lifts open items, elevated adds shadows.
To Tokens
Control tabs appearance: highlight adds a sliding pill, line shows an underline indicator, compact sizes tabs to content, rounded curves edges, panel-bg adds panel background, noicons hides icons.
Hover States
Use from-hover for accordion hover effects and to-hover for tabs hover. Both support bg (background color) and accent (text color) tokens. Mix multiple tokens for rich interactions.
Features
Responsive transformation, semantic HTML with details/summary elements, Container Queries for viewport-aware styling, and Anchor Positioning for the sliding tab indicator.
Styling
Highly customizable with token-based attributes. Apply multiple tokens to achieve different visual styles. All colors adapt to light and dark color schemes automatically.
Accessibility
Built on native details/summary elements for keyboard navigation and screen reader support. Focus management and ARIA states are handled natively by the browser.
Browser Support
Requires modern CSS features: Container Queries, Anchor Positioning, CSS Scope, Nesting, :has(), and :where() selectors. Check browser compatibility before use.
